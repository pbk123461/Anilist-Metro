Auth & onboarding
OAuth2 implicit flow (redirect to anilist.co/api/v2/oauth/authorize, capture token from redirect)
Token storage (secure keychain / encrypted prefs)
Token expiry detection (tokens live 1 year; prompt re-auth on expiry)
Guest / unauthenticated mode (full browse, no list or social write access)
Onboarding: explain features, prompt login, show sample data in guest mode
Deep-link support: open media/character/staff/user pages from external links
Pagination & API hygiene
All list queries use Page type with pageInfo (total, currentPage, lastPage, hasNextPage, perPage)
Infinite scroll or load-more pagination on all paged content
Rate limit handling: respect X-RateLimit-Remaining and X-RateLimit-Reset headers (90 req/min limit)
Retry with backoff on 429 Too Many Requests
MediaListCollection limited to 11,000 most recently updated entries — document this to user
Error boundary: surface GraphQL errors gracefully (errors array in response)
Optimistic UI updates for list mutations (update local state before server confirms)
Offline cache: serve stale list data when offline
